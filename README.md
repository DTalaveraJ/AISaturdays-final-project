# Tu IAbuela de Confianza

**Intelligent consumption optimization for Spanish households** — scrapes real supermarket prices, stores them in a local database, and runs a two-agent pipeline to build the cheapest grocery basket and plan the optimal shopping route.

Built as a project for [AISaturdays](https://www.aisaturdays.com/) Madrid.

---

## What this project does

1. **Fetches real product data** from Mercadona, Alcampo, and Dia APIs, normalizes prices (including IVA and discounts), and stores everything in a local SQLite database.
2. **Optimizes the grocery basket** using a greedy Python optimizer that selects the cheapest products covering every requested category, within a maximum number of shops.
3. **Plans a circular shopping route** using an LLM agent that calls OpenRouteService to find the optimal visit order starting and ending at the client's home. If the route exceeds the user's time or distance limit, it drops shops one by one until it fits — or suggests closest/cheapest single-shop alternatives.
4. **Opens the route in the browser automatically** after each run, with stops and API key pre-loaded and the route calculated instantly.
5. **Exposes a standalone web optimizer** (`iabuela-route-optimizer.html`) for interactive route planning directly in the browser.

## Components

### Multi-agent optimizer

Two implementations are available with identical behaviour and console output:

| File | Backend | When to use |
|------|---------|-------------|
| `agents_v2.py` | LangChain + LangGraph | Recommended — proper agentic tool-use loop, cleaner state management |
| `agents.py` | Ollama SDK directly | Fallback if LangChain is not installed |

#### `agents_v2.py` (LangChain + LangGraph)

```bash
pip install langchain-ollama langgraph langchain-core
ollama pull qwen2.5-coder:7b-instruct-q4_K_M
python agents_v2.py
```

Uses a `StateGraph` with four nodes wired in sequence:

```
build_basket → prepare_route → route_llm ↔ route_tools (loop) → finalize_route
```

The `route_llm ↔ route_tools` loop is the key LangGraph pattern: the LLM requests tool calls, the graph executes them, feeds results back, and keeps looping until the LLM emits a plain-text final answer — no manual `while True` needed.

#### `agents.py` (Ollama SDK)

```bash
pip install ollama
ollama pull qwen2.5-coder:7b-instruct-q4_K_M
python agents.py
```

Implements the same tool-use loop manually, with a text-fallback parser for models that emit JSON tool calls inside message content instead of using the native `tool_calls` field.

### Agent 1 — Shopping Optimizer (both versions)

Given a budget, list of food categories, number of people, and maximum number of shops:
- Queries `iabuela_catalog.db` for the cheapest product in each category.
- All categories are always included — coverage is mandatory even if the total exceeds the budget.
- Consolidates products into the fewest shops possible (up to `max_shops`) using a greedy pass that prioritises rarest-candidate categories first.
- Quantities scale automatically: 1 per person for cheap perishables (< 3 €), 1 unit otherwise.

### Agent 2 — Route Optimizer (both versions)

Given the shops selected by Agent 1 and a distance/duration limit:
- Geocodes the client's home address via OpenRouteService.
- Solves the Vehicle Routing Problem (VRP) to find the optimal circular route: home → shops → home.
- If the route exceeds the time or distance limit, drops shops one by one until it fits.
- If even the closest single shop exceeds the limit, uses the ORS Matrix API to rank all shops by round-trip travel time and suggests two alternatives:
  - **Option A — Closest**: shop with the lowest round-trip travel time.
  - **Option B — Cheapest**: shop with the lowest basket cost.

### Orchestrator — configurable inputs

| Parameter | Description |
|-----------|-------------|
| `budget` | Soft spending target in € |
| `categories` | List of food categories to cover (e.g. `["leche", "pan", "aceite"]`) |
| `n_people` | Household size — scales quantities |
| `max_shops` | Maximum number of different shops to visit |
| `home_address` | Client's starting and ending point |
| `transport_mode` | `driving-car`, `cycling-regular`, or `foot-walking` |
| `max_distance_km` | Hard route distance limit |
| `max_duration_min` | Hard route duration limit |

The ORS API key is read automatically from `my_apikey.json`.

After both agents finish, the route optimizer opens automatically in the browser with all stops and the API key pre-filled and the route already calculated.

To change the browser, set `BROWSER` at the top of the file:

```python
BROWSER = "chrome"   # "chrome" | "firefox" | "edge" | "" = system default
```

### Route Optimizer web app (`iabuela-route-optimizer.html`)

A single-file web app — no build step, no dependencies to install.

**Launched automatically** by the agents after each run — stops, transport mode, and API key are pre-loaded from `route_session.js` and the route is calculated immediately.

**Standalone use:**
1. Get a free API key at https://openrouteservice.org/dev/#/login
2. Open `iabuela-route-optimizer.html` in any modern browser
3. Paste your key, add addresses, choose a transport mode, click **Calcular Ruta Óptima**

The app calls the public ORS API. To use the self-hosted fork instead, change `ORS` in the script to `http://localhost:8080/ors`.

### Catalog database (`create_catalog_db.py`)

Creates `iabuela_catalog.db` with two tables and no external API calls:

| Table | Columns |
|-------|---------|
| `shops` | `shop_id`, `name`, `chain`, `address`, `latitude`, `longitude`, `city`, `postal_code` |
| `products` | `product_id`, `name`, `sku`, `price`, `shop_id` |

Seeded with 5 Madrid shops (Mercadona, Alcampo, Dia, Carrefour, Lidl) and 20 common grocery products repeated across all shops with chain-specific price multipliers.

```bash
python create_catalog_db.py
```

---

### Data pipeline (`lA_abuela.py`)

Scrapes and normalizes product catalogs from real supermarket APIs:

| Supermarket | API endpoint used |
|-------------|------------------|
| Mercadona   | `tienda.mercadona.es/api/categories/` |
| Alcampo     | Compra online product endpoints |
| Dia         | Analytics and product page endpoints |

```bash
python lA_abuela.py   # creates/updates iabuela_mvp.db
```

---

### Self-hosted routing engine (`openrouteservice/`)

A fork of [GIScience/openrouteservice](https://github.com/GIScience/openrouteservice) — a Java/Spring Boot service providing routing, geocoding, and VRP optimization APIs.

```bash
cd openrouteservice
docker-compose up
```

The service starts on `http://localhost:8080`. Requires an OSM extract (`.pbf` file) to build its routing graph. See [`openrouteservice/README.md`](openrouteservice/README.md) for full setup.

## Quick start

**Browser route optimizer only (no Python needed):**
```
1. Get a free key at https://openrouteservice.org/dev/#/login
2. Open iabuela-route-optimizer.html in any browser
3. Paste your key, add addresses, click Calcular Ruta Óptima
```

**Full agent pipeline (recommended — LangChain version):**
```bash
# 1. Install dependencies
pip install langchain-ollama langgraph langchain-core
ollama pull qwen2.5-coder:7b-instruct-q4_K_M

# 2. Create the catalog database (dummy data, no API needed)
python create_catalog_db.py

# 3. Place your ORS API key in my_apikey.json
#    File must contain a plain JSON string: "your_key_here"

# 4. Run
python agents_v2.py
```

**To use real supermarket data instead of dummy data:**
```bash
python lA_abuela.py   # populates iabuela_mvp.db
# then set DB_PATH = "iabuela_mvp.db" at the top of agents_v2.py
```
